## v8.7 — 입력 UI 개선 (2026-07-20, 배포 대기)
- **Enter = 줄바꿈**: 하단 빠른 추가 입력줄과 목록 내 수정칸에서 Enter를 눌러도 등록되지 않고 줄바꿈됨 (모바일 키보드 오등록 방지)

  - 윗줄: 체크박스 + 제목(전체 폭, 줄바꿈 표시)
  - 아랫줄: 중요도 점 · 폴더 · 마감일 · ✎ · ✕ 를 작은 정보줄로 배치
- 설명서 문구 갱신

## v8.6 — 빠른 추가 · 목록 내 수정 · 저장 확인 (2026-07-20, 배포 대기)
- **하단 빠른 추가 입력줄** (To-do · 메모 탭): ＋ 버튼 없이 화면 아래 입력줄에서 바로 추가
  - Enter 추가 · Shift+Enter/Ctrl+Enter 줄바꿈, 📁 버튼으로 저장 폴더 선택
  - 폴더를 선택하지 않으면 ★ 기본 폴더로, 기본 폴더가 없으면 [미지정]으로 저장
  - 메모는 첫 줄이 제목, 나머지 줄은 본문으로 저장
  - 달력 탭 외에는 ＋ 버튼이 숨겨지고 입력줄이 그 역할을 대신함 (할 일의 마감일·중요도는 추가 후 ✎에서 지정)
- **To-do 목록에서 바로 수정**: 제목을 탭하면 팝업 없이 그 자리에서 수정 — Enter 저장 · Shift/Ctrl+Enter 줄바꿈 · Esc 취소. 상세(폴더·마감일·중요도)는 기존 ✎ 버튼
- **할 일 여러 줄 입력**: 추가·수정 입력칸이 여러 줄 입력 가능한 칸으로 바뀌고, 목록에서도 줄바꿈 그대로 표시
- **할 일 하위 메모에 체크박스**: 세부 단계 완료 표시 가능 (할 일 전용)
- **메모 폴더 선택**: 에디터 제목 아래 폴더 칩으로 작성·수정 중 언제든 폴더 지정/변경
- **메모 저장 방식 변경**: 입력 즉시 자동 저장 → ✓ 버튼 저장 방식. 닫을 때 변경사항이 있으면 [저장/저장 안 함] 확인. 제목·내용이 모두 비어 있으면(공백 포함) 저장되지 않음
- **일정·할 일 작성 중 닫기 확인**: 일정 추가/수정, 할 일 추가/수정 시트를 작성 중에 닫으면(배경 탭·뒤로가기 포함) [저장/저장 안 함] 확인
- **메모 목록 ✕ 삭제 버튼**: 에디터를 열지 않고 목록에서 바로 휴지통 이동
- **여러 개 선택 → 폴더 이동**: To-do·메모 [선택] 모드에 [이동] 버튼 추가
- **입력 보존**: 하단 입력줄·목록 내 수정칸·하위 메모 입력칸은 동기화 수신 등으로 화면이 다시 그려져도 입력 중이던 내용과 커서가 유지됨
- 알려진 확인 사항(원인 조사, 8번 문의): 앱에 구글 캘린더로 인한 **주기적 자동 새로고침 타이머는 없음**. 다만 Firestore 실시간 수신(다른 기기 변경 반영·공유 일정 갱신)과 구글 캘린더 불러오기 완료 시 화면 전체가 다시 그려지며, 이때 화면 안(목록 영역)의 입력칸이 비워질 수 있었음 — v8.6의 새 입력칸들에는 위 입력 보존을 적용, 나머지 전면 보존은 추후 버전에서 검토
- CACHE: `planner-v8-6`

## v8.3 — 구글 캘린더 자동 연동(쓰기) (2026-07-19, 배포 대기)
- 계정 시트에 **[내 일정을 구글 캘린더에 자동 등록]** 토글 추가 — 켜면 일정 추가·수정·삭제·드래그 이동이 로그인 계정의 구글 기본 캘린더에 자동 반영
- 반복 일정도 RRULE로 변환해 반영(매일/매주 요일/매월 날짜·N째 요일/매년 + 간격·종료일, 개별 제외는 EXDATE)
- [이 일정만] 수정·삭제·이동도 그대로 반영(시리즈 EXDATE + 분리 일정 신규 등록)
- 메모형·체크리스트형 일정은 종일 일정(제목)으로 등록
- 토글 켤 때 기존 일정 일괄 업로드 선택 가능([모두 올리기])
- 앱에서 올린 일정은 [구글 캘린더 불러오기] 시 자동 제외되어 중복 표시 없음
- 오프라인·팝업 차단 등으로 실패한 변경은 대기열에 보관 → 계정 시트 [⤴ 지금 동기화]로 재시도
- 쓰기 권한 토큰은 약 50분 캐시 — 만료 후 첫 반영 시 구글 팝업이 잠깐 떴다 닫힐 수 있음(같은 계정 자동 진행). 권한 범위가 읽기 전용→일정 수정으로 확대되어 최초 1회 동의 필요
- 토글을 꺼도 이미 올라간 일정은 구글에 남음(연동만 중지)
- CACHE: `planner-v8-3`

## 검증(증적) 방법
1. 각 버전의 원본 코드는 커밋 SHA로 대조 가능: `https://github.com/hye1w/DOLIST/commit/<SHA>`
2. 매 버전 배포 전 Node 스텁(DOM 목킹) 스모크 테스트 수행: index.html에서 스크립트를 추출해 초기 렌더 및 주요 흐름 검증
3. 배포 후 앱 2회 재실행 → 👤 계정 시트 하단 버전 표시로 적용 확인
4. index.html의 `APP_VERSION`과 sw.js의 `CACHE`는 항상 쌍으로 갱신(예: v8.0 / planner-v8-0)
